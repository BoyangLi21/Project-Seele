# EVA Unit-01 暴走人工测试

暴走是服务端状态机，不是只播放一段动画。只有初号机在驾驶状态下同时满足“低生命、断电、高同步”才会夺走控制权；驾驶员被弹出后，初号机只追击 `Angel` 标记实体，结束后进入独立的五分钟强制停机。

## 当前默认值（新增，待玩家拍板）

| 项目 | 默认值 | 配置键 |
|---|---:|---|
| 机体生命触发线 | ≤15% | `eva_berserk.healthThreshold` |
| 驾驶员同步率触发线 | ≥60.0% | `eva_berserk.syncThreshold` |
| 自主阶段 | 900 tick（45 秒） | `eva_berserk.durationTicks` |
| 强制停机 | 6000 tick（5 分钟） | `eva_berserk.recoveryTicks` |
| 爪击伤害倍率 | ×2.5（默认每击 50） | `eva_berserk.damageMultiplier` |
| 使徒索敌范围 | 128 格 | `eva_berserk.targetRange` |

没有改变普通驾驶近战、武器、使徒或 A.T. 力场数值。暴走爪击仍属于 EVA 近战，因此能削减使徒 A.T. 力场；不会像阳电子炮那样穿透完整力场。

## 快速触发

1. 在远离脐带供电桩的开阔地召唤 Unit-01 和一只使徒并登乘初号机。
2. 执行 `/seele pilot sync set 80`。
3. 对最近初号机执行：

```mcfunction
/data merge entity @e[type=projectseele:eva_unit01,limit=1,sort=nearest] {Health:30.0f,SeelePowerTicks:0}
```

默认最大生命 300，30 低于 15% 触发线。若测试配置改过最大生命，请把 `Health` 调到最大生命的 15% 以下。

## 验收点

1. 条件齐备后的下一个服务器 tick，驾驶 HUD 消失，玩家在机体脚边安全弹出并短暂获得抗性/缓降；聊天出现“初号机控制丧失”。
2. 初号机切为空手、关闭 A.T. 力场和所有蓄力，发出咆哮并出现头部红色眼部粒子；脐带必须断开，电量保持 0。
3. 128 格内同时放置使徒、玩家、牛、另一台 EVA。初号机只能把使徒设为目标，不得追逐或伤害其他对象。
4. 初号机应导航到最近使徒，左右爪交替，默认每 10 tick 最多攻击一次、基础 20 × 2.5 = 50 伤害。使徒完整 A.T. 力场先掉力场耐久，不能直接掉本体血。
5. 900 tick 后自主阶段停止、速度归零，NERV 机体屏从红色 `BERSERK / AUTONOMOUS` 切到金色 `FORCED SHUTDOWN`。
6. 强制停机期间右键插入栓必须被拒绝并显示剩余秒数；即使把机体推回脐带桩、内部电池恢复满电，也不能绕过五分钟停机。
7. 暴走中退出并重启存档，剩余自主 tick、目标重搜和最终停机必须继续；停机中重启也不能清空剩余时间。
8. 对 Unit-00/02 重复低血、断电、80% 同步条件；两者不得进入初号机专属暴走。
9. 生命树束缚或发射井锁定期间不得触发；不能让暴走中断载台或逃离十字演出。

## 离线门禁

```bat
python tools\validate_eva_berserk_contract.py
```

门禁检查触发四条件、驾驶员安全弹出、使徒专属索敌、断电越权、爪击/A.T. 力场语义、动画与红眼、跨重启状态、五分钟登乘锁和指挥室倒计时。实际追路、咆哮姿态、红眼位置和战斗重量仍需人类在游戏里验收。
