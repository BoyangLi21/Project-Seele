# EVA 驾驶员同步率人工测试

同步率现在属于玩家，而不是按 Unit-00/01/02 写死的装饰数字。数据通过 Forge Player Capability 持久化；进入任一 EVA 后，服务器把同一权威值同步到机体、驾驶 HUD 和 NERV 指挥室。机体受损、A.T. 力场负载、运动和阳电子炮充能仍会在 HUD 数字上造成短时波动，但成长基值保存在驾驶员身上。

## 当前默认值（新增，待玩家拍板）

| 项目 | 默认值 | 配置键 |
|---|---:|---|
| 初始同步率 | 40.0% | `eva_synchronization.initial` |
| 上限 | 100.0% | `eva_synchronization.maximum` |
| 有效驾驶成长 | 每分钟 +0.25% | `eva_synchronization.driveGainPerMinute` |
| 驾驶 EVA 击杀使徒 | +2.5% | `eva_synchronization.angelKillGain` |
| 满同步移动加成 | +25% | `eva_synchronization.maxMobilityBonus` |
| 满同步攻击响应 | +25% | `eva_synchronization.maxAttackSpeedBonus` |
| 神经反噬起点 | 60.0% | `eva_synchronization.feedbackThreshold` |
| 满同步反噬比例 | 实际机体伤害的 35% | `eva_synchronization.maxFeedbackFraction` |

攻击响应只缩短空手/粒子刀/朗基努斯近战、重击、踩踏和帕雷特步枪的服务器冷却；不改变每击伤害，不缩短阳电子炮蓄力或战略爆炸冷却。移动加成作用于站立、奔跑、跪姿和卧姿。

## 快速命令

- `/seele pilot sync`：查看自己的持久化基值。
- `/seele pilot sync set 40`：恢复默认基准。
- `/seele pilot sync set 80`：快速验证高同步响应和反噬。
- `/seele pilot sync set 100`：验证上限响应。

这些命令要求 2 级权限，仅用于开发与管理员验收。

## 单人验收

1. 新玩家执行 `/seele pilot sync`，应显示 `40.0%`。登乘任何一台 EVA，启动完成后顶部读数应围绕 40% 轻微波动，而不再因机型自动变成 38.2/41.3/52.4。
2. 保持 EVA 有电、非发射锁定、非生命树束缚并实际驾驶 60 秒；再次查询应增加 0.25%。中途退出后进度保留，下一次驾驶接着累计。
3. 以 EVA 对使徒造成最后一击；应收到 A10 稳定提示，基值增加 2.5%。普通生物、下车后击杀、由其他玩家完成最后一击都不得奖励。
4. 分别在 40% 和 100% 直线行走/跑步并连续攻击。100% 时移动速度最多提高 25%，近战与帕雷特步枪服务器冷却最多缩短 20%（速度倍率 1.25 的倒数），伤害数字不变。
5. 设为 80%，关闭 A.T. 力场，让使徒实际打掉 EVA 机体生命。只有真实穿透到机体的伤害才产生驾驶员反噬；80% 时目标比例约为实际机体伤害的 17.5%。完全被 A.T. 力场吸收的攻击不得伤害驾驶员。
6. 设为 100%，确认反噬上限为实际机体伤害的 35%；玩家仍受原版无敌帧、护甲、难度等最终结算影响。
7. 退出世界重进、跨维度、正常死亡重生后再次查询；同步率和不足一分钟的驾驶累计都必须保留。
8. 一名玩家驾驶、一名玩家留在 NERV Operations。驾驶 HUD 与对应机体屏的 `SYNCHRO` 应显示相同的实时读数，不能由客户端自行猜测。

## 离线门禁

```bat
python tools\validate_eva_sync_contract.py
```

门禁检查 Capability 注册/附着/克隆、NBT 持久化、驾驶与击杀成长、机体同步数据、响应倍率、实际机体伤害反噬、管理员命令以及 HUD/指挥室共用读数。它不能替代多人实机对移动手感、成长速度与高同步风险的判断。
