# NERV 发射井测试

当前实现是一套可重复验证的三井出击原型，三座井分别部署零号机、初号机和二号机。井底有 13×13 固定平台，背部有高位插入栓栈桥；同步序列结束后，独立的 11×11 移动载台会把 EVA 弹射到地表。

## 不启动 Minecraft 的结构验证

先运行离线预览：

```bat
python tools\render_launch_silo_preview.py --strict
```

脚本直接读取 `NervConstructionKitItem`、`LaunchSiloCommands`、`EvaUnit01Entity`、初号机渲染比例、初号机几何与 `activation` 动画，不另维护一套手填尺寸。默认生成：

- `external-assets/work/launch-silo-preview/launch_silo_preview.png`
- `external-assets/work/launch-silo-preview/launch_silo_preview.json`

PNG 同时给出三井俯视图和中井剖面图，明确标出：

- 三个型号及各自井床；
- 井壁、11×11×31 上升净空、13×13 固定井底平台与 11×11 移动载台；
- 地表、表面闸板、固定井底平台和移动载台的 31 格行程；
- 与本机下载并缩放适配的插入栓最终位置对齐的背部高位栈桥、24 格连续梯道和登乘点；
- 插入栓从高处下降至背部插口的动画路径；
- EVA 弹射终点（井床上方 32 格）。

`--strict` 会校验三型号、三床位、井间距、井壳、11×11×31 净空、EVA 宽度、13×13 固定井底平台、11×11 移动载台、高位栈桥、梯道连续性、背部登乘扇区、栈桥与运行时插入栓位置、插入栓自上而下、31 格载台行程与弹射终点。任意 Java 循环、常量或动画合同漂移都会在 JSON 中留下 `FAIL` 并返回非零退出码。

这个离线图是尺寸和状态合同检查，不是游戏画面替代品。它不能证明 Minecraft 中的碰撞、插值、粒子、声音和弹射重量感已经通过。

## 一键布置与弹射

1. 用 `tools/start_test.bat` 启动开发客户端。
2. 进入允许作弊的创造模式世界，在开阔地表执行 `/seele silo setup`。
3. 命令会建造三井设施，并把玩家放到初号机背部的高位插入栓栈桥。
4. 面向初号机后颈下方的插入栓入口并按右键；即使入口网格超出粗略碰撞箱，专用交互射线也会把请求交给服务器重新验证。`/seele silo board` 只作为确定性测试后备。
5. 插入栓锁定、LCL 与同步演出结束后，11×11 实体移动载台托住 EVA 自动弹射到地表。
6. 任意时刻执行 `/seele silo status` 查看 `LOCKED`、`ASCENT`、`SURFACE_CLEAR` 或 `IDLE`，以及实时 `carrier` 高度。
7. 弹射前执行 `/seele silo audit`；只有结果同时显示 `units=3`、`variants00/01/02=true`、`beds=3`、`highGantries=3` 与 `clearShafts=3`，才表示游戏内三井结构门禁通过。

创造物品栏里的 `NERV 出击设施建造器` 会生成相同设施。所有 EVA 都从实际显示的背部插入栓入口登乘：玩家必须位于约 24–29.5 格相对高度、0.75–9.5 格水平距离和背部扇区内，并真正把准星对准入口；从正面、侧面或脚边隔空右键会被拒绝。三份 Tiger OBJ 本身没有独立的插入栓子网格；当前栓体取自用户下载的 EVA-02 评估模型并重新缩放、换轴和绑定，舱门则是为 Tiger 背部插口单独制作的适配几何，两者都只用于本机评估。

## 游戏内验收点

- 地下横向通道的梯井可通往背部高位栈桥；`audit` 会逐格检查 24 格连续梯子。
- 从井底或 EVA 脚边右键应提示前往高位栈桥，不应直接进入驾驶室。
- 登乘后播放六秒插入栓/同步覆盖层；栓体从上方下降并完全收入背部，落座后外部只应看到闭合舱门，不能有半截栓体漏在装甲外，也不能在下降途中突兀跳切；约五秒时进入最终联锁并启动载台。
- 启动与井底联锁期间，机体位置、重力、移动输入、碰撞推动和机体朝向全部冻结；转动驾驶员视角不能带动 EVA 在井内扭转。
- 上升期间 11×11 移动载台随 EVA 逐层移动，EVA 被引导到井中心；井底保留独立的 13×13 固定平台，驾驶输入和 `V` 弹出被联锁。
- 中途异常终止会清掉井内移动载台并重新关闭地表载台，不在井筒内遗留方块层。
- 中断时机体先回井底安全位，乘员随座舱一起复位；正常完成则先让机体与乘员整体越过 deck 平面，再闭合闸门。
- `LOCKED`/`ASCENT` 中保存并重进后有 40 tick 乘员关系恢复窗口；日志应出现 `NERV carrier recovered`，井内只能重建一层完整载台。
- EVA 基座到达井床上方 32 格后制动；随后 `LAUNCH_CLEAR` 保持 18 tick（约 0.9 秒）的地表安全联锁，期间仍冻结输入，联锁结束后才恢复重力与驾驶权。
- `run/logs/latest.log` 先出现 `NERV silo audit [setup]: valid=true`，随后依次出现 `NERV launch locked`、`NERV launch ascent`、`NERV launch surface clear`。

## 启动器参数

需要无人值守的真实发射井演出批次时，可从命令行运行：

```bat
tools\start_test.bat visual silo
```

该入口会真实执行 `setup` 与 `board`，而不是把 EVA 瞬移到几个假定位置。客户端依据同步后的 `IDLE → LOCKED → ASCENT → SURFACE_CLEAR` 状态及插入栓倒计时，依次保存六张 PNG：高位后背平台、插入栓下降外景、同步驾驶舱、舱盖闭锁、井内中段上升和地表释放。任一阶段漏失、超时、型号网格回退或截图失败都会令批次返回非零。

固定测试会把专用世界重置为晴天正午，避免持久化的午夜或雷雨掩盖机械结构。高位平台的梯井顶部保留真实攀爬开口；未启动 EVA 的实体朝向、身体朝向和头部朝向也会同时锁定，确保第一张图从平台看到后背插入栓舱盖，而不是面部。截图保存在 `run/screenshots/projectseele_visual/<批次时间>/`。

桌面启动器会先重建并校验本地高细节资源，再启动 Forge。自动批次能证明实际状态机和固定机位画面连续存在；井壁碰撞的操作手感、载台重量感与镜头舒适度仍需人类查看生成的六张图或亲自驾驶后作最终结论。
