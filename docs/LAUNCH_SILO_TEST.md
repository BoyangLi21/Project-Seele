# NERV 发射井测试

当前实现是可重复验证的原型流程，三座井分别部署零号机、初号机和二号机。

## 一键布置与弹射

1. 从 `tools/start_test.bat` 启动开发客户端。
2. 进入一个允许作弊的创造模式世界，在开阔地表执行 `/seele silo setup`。
3. 命令会建造三井设施，并把玩家放到初号机背部的高位插入栓栈桥。
4. 执行 `/seele silo board`。插入栓锁定、LCL 与同步演出结束后，11×11 实体载台托住 EVA 自动弹射到地表。
5. 任意时刻可执行 `/seele silo status` 查看 `LOCKED`、`ASCENT`、`SURFACE_CLEAR` 或 `IDLE`，以及实时 `carrier` 高度。
6. 弹射前执行 `/seele silo audit`；只有结果同时显示 `units=3`、`variants00/01/02=true`、`beds=3`、`highGantries=3` 与 `clearShafts=3`，才表示三井结构门禁通过。每个 `clearShafts` 会检查可容纳 8.5 格宽 EVA 的 11×11×31 格上升净空。

创造物品栏里的 `NERV 出击设施建造器` 也会生成同一套设施。井内 EVA 只能从背部高位栈桥进入：玩家必须同时处于允许高度、4–11 格距离和机体背部扇区；从正面或侧面隔空右键会被拒绝。地面自由放置的 EVA 仍保留直接驾驶方式。

## 验收点

- 地下横向通道的梯井能够通往背部高位栈桥；`audit` 会逐格检查 20 格连续梯子，不再只检查首尾。
- 从井底或 EVA 脚边右键会提示前往高位栈桥，不会直接进入驾驶室。
- 登乘后播放六秒插入栓/同步覆盖层；约五秒时进入最终联锁并启动载台。
- 上升期间 11×11 载台随 EVA 逐层移动，EVA 被引导到井中心；驾驶输入和 `V` 弹出被联锁，不能撞壁或把驾驶员丢在井中。
- 中途异常终止会清掉井内移动载台并重新关闭地表载台，不在井筒内遗留方块层。
- 中断时机体会先回到井底安全位，乘员随座舱一起复位，再关闭地表载台；正常完成则先把机体/乘员整体移出 deck 平面再闭合，闸门不得穿过机体。
- 上升联锁期间原版 Shift 下机与跨维度传送都会被拒绝。同一 lodestone 只能由一台活动 EVA 占用，第二台会收到占用提示。
- 将井底 EVA 水平推移 1–4 格后，从地面右键仍必须被高位门禁拒绝；背部栈桥入口仍应成功。发射台识别检查完整 11×11 载台范围，不再只看脚下单列。
- 在 `ASCENT` 中保存并重进后，日志应出现 `NERV carrier recovered`；井内只能重建一层完整载台，不得留下半层或重复高度。
- EVA 基座到达地表上方两格后制动并恢复重力与驾驶权。
- `run/logs/latest.log` 先出现 `NERV silo audit [setup]: valid=true`，随后依次出现 `NERV launch locked`、`NERV launch ascent`、`NERV launch surface clear`。

## 启动器参数

双击启动器仍是普通测试模式。需要无人值守的初号机截图批次时，可从命令行运行：

```bat
tools\start_test.bat visual
```
