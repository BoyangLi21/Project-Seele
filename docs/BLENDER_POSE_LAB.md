# Project SEELE 初号机 Blender 动作台使用手册

这套动作台的目的不是在 Blender 里重新建模，而是让人类能直接看到并调整
Minecraft 实际使用的初号机骨架。它读取本机 `eva_real_model` 资源包里的：

- 初号机三角网格与 1024×512 贴图；
- GeckoLib/Bedrock 36 根骨骼及父子关系；
- 当前 `eva_unit01.animation.json` 动作；
- 阳电子炮、粒子刀、朗基努斯之枪；
- 插入栓与背部舱盖；
- 驾驶员视角的同一套身体网格。

导入与导出已经包含游戏渲染器使用的 X 反射和 X/Y 旋转反号。不要再手工把
整个模型旋转 90° 或镜像；那会让 Blender 看似正确、进游戏后再次反向。

## 1. 正确启动方式

双击：

```text
D:\eva\tools\start_pose_lab.bat
```

第一次启动如果动作台还不存在，脚本会先在后台构建：

```text
D:\eva\external-assets\work\pose-lab\Project_SEELE_Unit01_PoseLab.blend
```

以后不要直接双击 `.blend`。直接双击虽然能看到模型，但不会注册右侧的
`Project SEELE` 工具面板。始终从 `start_pose_lab.bat` 进入。

动作台、参考图、截图和导出的 JSON 全在 `external-assets/` 中，已经被 Git
忽略，不会误提交第三方模型或参考图。

### 不要随便运行重建脚本

`D:\eva\tools\rebuild_pose_lab.bat` 会重新生成固定的 `.blend`，因此会覆盖尚未
导出的人工姿势。只有以下情况才运行它：

1. Codex 修改了导入器或骨架结构并明确让你重建；
2. `.blend` 已经损坏；
3. 你的重要姿势都已点过“导出当前动作 JSON”。

## 2. 第一次进入 Blender 后

1. 把鼠标放到中间的 3D 视图。
2. 按 `N`，打开右侧侧栏。
3. 在侧栏顶部点击 `Project SEELE` 标签。
4. 你应该看到“Project SEELE — 初号机动作台”。
5. 点击“正”机位，确认整台初号机面向你、脚底在地面线上。

如果没看到面板，说明你是直接打开了 `.blend`；关闭 Blender，然后用
`start_pose_lab.bat` 重开。

## 3. Blender 界面中只需要认识的部分

### 3.1 3D 视图

中间最大的区域。鼠标操作：

- 中键拖动：环绕观察；
- `Shift + 中键`：平移；
- 滚轮：缩放；
- 小键盘 `1`：正视图；
- 小键盘 `3`：右侧视图；
- 小键盘 `7`：顶视图；
- 小键盘 `5`：透视/正交切换；
- 小键盘 `0`：当前固定摄像机；
- 选中骨骼后按小键盘 `.`：把视图聚焦到该骨骼；
- `Home`：把全部可见物体纳入视野。

没有独立小键盘时，直接使用 Project SEELE 面板里的“正 / 侧 / 背 / 3/4 /
驾驶员”按钮即可。

### 3.2 Outliner（右上角对象列表）

动作台按用途分为：

- `01_BODY`：初号机身体；
- `02_ATTACHMENTS`：武器、插入栓、舱盖；
- `03_REFERENCES`：人类单膝跪与卧姿射击参考图；
- `04_CAMERAS`：固定验收机位；
- `05_ENVIRONMENT`：地面和方向轴；
- `SEELE_Unit01_Rig`：必须调整的唯一骨架。

不要进入身体网格的 Edit Mode，不要拖网格顶点。当前工作只在骨架的
Pose Mode 中做。

### 3.3 模式选择

左上角会显示 `Object Mode`、`Edit Mode` 或 `Pose Mode`。

- `Object Mode`：选择整个物体；
- `Pose Mode`：调整骨骼，应该使用这个；
- `Edit Mode`：修改骨架结构，本轮禁止使用。

点击面板里的任一“选择关节组”按钮，会自动选择骨架并进入 Pose Mode。

## 4. 最重要的左右与坐标规则

### 4.1 `_l` 和 `_r` 是 EVA 自己的左右

- `_l` = EVA 自己的左侧；
- `_r` = EVA 自己的右侧；
- 从正面看 EVA 时，`_l` 会出现在屏幕右边；
- 从正面看 EVA 时，`_r` 会出现在屏幕左边。

这是最容易再次把左右手做反的地方。不要按屏幕左右猜骨骼名。

### 4.2 局部轴

每根骨骼使用与游戏一致的局部轴：

- 红色 `X`：左右轴；绕 X 转通常是人体关节的前后屈伸；
- 绿色 `Y`：上下轴；绕 Y 转通常改变肢体朝向/内外旋；
- 蓝色 `Z`：前后轴；绕 Z 转通常是向身体内侧或外侧张开。

在 Blender 顶部把 Transform Orientation 设为 `Local`，就能用彩色旋转环直接
拖局部轴。

键盘精确旋转时：

- `R`、`X`、`X`：绕骨骼自己的 X 轴；
- `R`、`Y`、`Y`：绕骨骼自己的 Y 轴；
- `R`、`Z`、`Z`：绕骨骼自己的 Z 轴；
- 输入数字后回车，例如 `R X X 5 Enter`；
- 输入负数，例如 `R X X -5 Enter`。

按两次轴键很重要：一次通常是全局轴，两次才是骨骼局部轴。

### 4.3 推荐调整步长

- 肩/髋：每次 2°–5°；
- 肘/膝：每次 3°–8°；
- 手腕/脚踝：每次 1°–4°；
- 手指：每次 3°–6°；
- 武器握点平移：每次 0.02–0.10 Blender 单位。

不要一次旋转 30° 后再猜。小步调整、轮流看正面和侧面，速度反而更快。

## 5. 骨骼名称表

### 躯干与头

| 骨骼 | 用途 |
|---|---|
| `root` | 整体上下/前后位移；最后才调 |
| `torso_lower` | 骨盆与腰部整体倾斜 |
| `torso_upper` | 胸腔倾斜与扭转 |
| `aim_pitch` | 瞄准俯仰的双肩公共层；不要拿它修单侧手臂 |
| `head` | 头部 |
| `horn` | 头角绑定倾斜；通常不要动 |
| `pylon_l/r` | 两侧肩部拘束器，随胸腔走 |

### 双臂

| EVA 左侧 | EVA 右侧 | 人体部位 |
|---|---|---|
| `arm_l` | `arm_r` | 上臂/肩 |
| `forearm_l` | `forearm_r` | 前臂/肘 |
| `hand_l` | `hand_r` | 手腕与掌部 |

### 双腿

| EVA 左侧 | EVA 右侧 | 人体部位 |
|---|---|---|
| `leg_l` | `leg_r` | 大腿/髋 |
| `shin_l` | `shin_r` | 小腿/膝 |
| `foot_l` | `foot_r` | 脚踝与脚掌 |

### 手指

每只手当前有五根整指骨骼：

- `finger_thumb_l/r`；
- `finger_index_l/r`；
- `finger_middle_l/r`；
- `finger_ring_l/r`；
- `finger_little_l/r`。

当前模型每根手指只有一根骨骼，所以可以做“整根手指弯曲”和握拳轮廓，不能
分别控制近节、中节、远节三个指关节。真正逐节握拳需要以后把手指网格切分并
增加约 20–30 根指骨。现在先把整指握点做对；这不是你不会操作，而是当前骨架
本身的上限。

在目前模型里，手指局部 X 的正方向通常是闭合。建议先只调 X：

- 轻搭：约 25°–45°；
- 半握：约 50°–70°；
- 握紧：约 75°–95°；
- 拇指通常比其余四指少弯约 15°–30°。

以上只是起点，最终以手掌是否穿过枪身、指尖是否落在握把上为准。

## 6. Project SEELE 面板各项功能

### 6.1 “载入当前游戏动作”

下拉选择动作，点击“载入姿势”。它会清空当前人工修改，然后从正式
`eva_unit01.animation.json` 重新取样。

因此：载入另一个姿势前，必须先“导出当前动作 JSON”。否则刚才的调整会丢失。

### 6.2 “选择需要调整的关节”

这些按钮只负责快速进入 Pose Mode 并选骨骼，不会改动作：

- 躯干/头；
- 双臂、左臂、右臂；
- 双腿；
- 十根手指；
- 武器/插入栓骨骼。

同时选中多根骨骼时旋转会让它们各自一起转。细调时最好点空白取消选择，再单击
一根骨骼，只让活动骨骼呈浅蓝色。

### 6.3 “武器”

只控制显示哪一种武器，不会自动改手臂：

- 徒手；
- 阳电子炮；
- 高振动粒子刀；
- 朗基努斯之枪。

姿势预设会自动选择相应武器，但你也可手动切换进行对比。

### 6.4 “显示插入栓与舱盖”

用于调整：

- `entry_plug`：插入栓；
- `plug_hatch_l`、`plug_hatch_r`：两片背部舱盖。

插入栓目前是独立附件。先用它校准真实入口位置、角度和进入深度，不要通过移动
整个初号机来迁就插入栓。

### 6.5 “显示人类姿势参考图”

打开后会显示用户提供的：

- 单膝跪姿结构图；
- 单膝跪姿照片；
- 卧姿射击照片。

参考图只在视口显示，不参与游戏导出，也不会提交 Git。

### 6.6 “驾驶员视角遮挡规则”

打开时隐藏会包住摄像机的头、胸腔、骨盆和肩部拘束器，但保留同一套真实手臂与
腿。它不是另做一对漂浮手臂，因此你调整第三人称骨骼后，驾驶员视角立刻同步。

切回第三人称验收前记得关闭这个选项，否则会误以为模型缺少身体。

### 6.7 固定机位

- 正：判断左右对称、枪口横向偏移、双膝是否向外翻；
- 侧：判断手臂是在身前还是背后、枪口是否指向前方、重心是否合理；
- 背：检查手臂是否穿入拘束器、左右是否误反；
- 3/4：检查整体轮廓和穿模；
- 驾驶员：检查第一人称和第三人称是否真正共用动作。

任何动作至少通过正面、侧面、3/4 三个视角才算可以交给游戏测试。

## 7. 通用调姿顺序

所有动作都按下面顺序，不能倒过来：

1. 载入最接近的当前姿势；
2. 先判断头的朝向。头角指向哪边，身体正面就在哪边；
3. 调 `root` 以外的大关节：骨盆、胸腔、髋、肩；
4. 调中关节：膝、肘；
5. 调末端：脚踝、手腕；
6. 调武器骨骼的方向和握点；
7. 调另一只手去接触武器；
8. 最后调十根手指；
9. 再调 `root` 的高度，让脚、膝、肘或胸部接触地面；
10. 正/侧/背/3/4 检查；
11. 驾驶员视角检查；
12. 导出 JSON；
13. 输出五视图 PNG；
14. 再切换到下一个姿势。

如果一开始就动 `root` 或手指，很容易用整体位移掩盖错误的膝肘方向。

## 8. 第一个练习：把单膝跪姿做成人类姿势

建议先只做单膝跪，因为它最容易验证骨架方向，也会为跪姿射击打基础。

### 8.1 建立工作副本

1. 在姿势下拉框选择“当前单膝跪 / Crouch”。
2. 点击“载入姿势”。
3. 打开“显示人类姿势参考图”。
4. 点击“侧”机位。
5. 先不要动任何手指或手臂。
6. 把导出动作名改成 `crouch_human_v1`。

建议让 EVA 自己的右膝落地、左脚在前。正面观察时，`leg_r` 在屏幕左边，
`leg_l` 在屏幕右边。

### 8.2 调落地腿（EVA 右腿）

物理目标：右大腿从髋部向后下方，右小腿折回，膝盖触地，髋—膝与膝—踝形成
接近直角的折线，脚背或前脚掌自然落在后方。

1. 点击“选择双腿”。
2. 取消其他选择，只选 `leg_r`。
3. 主要绕局部 X 小步旋转，使大腿向后下方，而不是把整条腿甩到身体侧面。
4. 只选 `shin_r`，同样主要绕局部 X，让小腿向后折；膝关节不能反折。
5. 只选 `foot_r`，让脚背/前脚掌与地面关系自然。
6. 每改一次就在“侧”和“3/4”之间切换。

若侧面看膝盖弯曲正确，但正面看整条腿向左/右飞出，说明 Z 轴外展过多；先把
`leg_r` 的局部 Z 减小，而不是反转 X。

### 8.3 调支撑腿（EVA 左腿）

物理目标：左脚完整踩地，左膝位于左脚上方或略后，左大腿承担身体重量。

1. 只选 `leg_l`，绕局部 X 把大腿送到身体前下方；
2. 只选 `shin_l`，让膝部形成自然弯曲，不要让小腿穿过大腿；
3. 只选 `foot_l`，让鞋底尽量平贴地面；
4. 正面看左膝不要跨过身体中线；
5. 侧面看左膝不要远远超过脚尖。

### 8.4 调骨盆、躯干与整体高度

1. 选 `torso_lower`，让骨盆略向前倾，而不是完全后仰；
2. 选 `torso_upper`，让胸腔保持在支撑面上方；
3. 头应看向前方，不应低头钻入胸甲；
4. 最后选 `root`，使用 `G Z` 只调整整体高度；
5. 让右膝接触地面，同时左脚底不穿地。

注意：`root` 的 Blender 全局 Z 是上下；骨骼局部轴说明只适用于旋转。整体高度
最好直接用移动工具的蓝色全局 Z 箭头。

### 8.5 手臂先放在合理待机位置

单膝跪的基础姿势不用先瞄准：

1. `arm_l/r` 应在胸腔前侧或身体两侧，不能被掰到背后；
2. `forearm_l/r` 保持轻微屈肘；
3. 正面看左右手不能交叉；
4. 侧面看手腕必须位于胸口前方，而不是背部拘束器后方。

完成后：

1. 点击“导出当前动作 JSON”；
2. 点击“输出五视图 PNG”；
3. 告诉 Codex 导出名 `crouch_human_v1`；
4. 不要立即重建动作台。

## 9. 卧姿射击与匍匐前进

目标是用户参考图中的贴地射击，不是四肢僵直的“地上游泳”，也不是狗一样高抬
骨盆的爬行。

### 9.1 卧姿射击静态关键帧

1. 载入“卧姿射击 / Prone Cannon”。
2. 打开阳电子炮。
3. 先用 `torso_lower` 让骨盆与胸腔接近水平；
4. 用 `torso_upper` 只做少量抬胸，给双臂留下空间；
5. `head` 抬到能沿枪口方向观察，不要反向折颈；
6. 双肘向身体两侧前方支撑，肘部应接近地面；
7. 炮托与右肩形成接触；
8. 左手位于炮身前部支撑点，不能穿过枪；
9. 双腿向后伸展，可略微分开；
10. 膝盖只有轻微弯曲，不能整条腿翻到身体前面；
11. 最后下移 `root`，让胸腹和肘部贴近地面。

### 9.2 匍匐循环的两帧

动作台提供 Crawl A（0.00 秒）和 Crawl B（0.70 秒）。正确低姿匍匐应满足：

- 胸腹始终贴近地面；
- 左肘前进时右膝/右腿提供推进，下一半周期反过来；
- 骨盆不能像四足动物一样高抬；
- 双腿不能同时被甩到身体正前方；
- 头和胸腔始终朝前；
- 手臂不能藏在背后。

先分别修 A、B 两个静态关键帧并导出。Codex 再把二者和中间过渡写回循环动画。

## 10. 阳电子炮双手瞄准

正确顺序：武器方向 → 主手 → 支撑手 → 肩胸 → 手指。

1. 载入“阳电子炮瞄准 / Cannon”。
2. 点击“侧”机位。
3. 先选 `cannon` 骨骼，调整旋转，使枪管沿 EVA 正前方，而不是横倒 90°；
4. 必要时用 `G X X / G Y Y / G Z Z` 小幅移动武器握点；
5. `hand_r` 对准主握把；
6. `forearm_r` 调整肘部，使手腕保持相对中立；
7. `arm_r` 把枪托送到右肩附近；
8. `hand_l` 落在前护木/支撑握点；
9. `forearm_l` 和 `arm_l` 再去迁就左手位置；
10. 正面检查枪口不要向左肩或右肩偏；
11. 顶视图检查枪管与身体朝向平行；
12. 最后弯曲手指包住握把，不能五指展开，也不能穿枪身。

两只手不需要完全对称。真实持枪中，主手负责扳机与握把，支撑手在更前方，两个
肘的位置和角度本来就不同。

## 11. 反握粒子刀

反握的判定不是“刀在手边”，而是：刀刃从小指一侧向手腕后下方延伸。

1. 载入“反握刀准备 / Knife Ready”。
2. 只显示粒子刀。
3. 先选 `knife`，把刀柄放入右掌；
4. 从侧面确认刀刃朝后/下，而不是从拇指一侧向前伸；
5. 调 `hand_r` 让手腕不过度折弯；
6. 调 `forearm_r`、`arm_r`，让刀手处在身体前侧可攻击的位置；
7. 四指围住刀柄；
8. 拇指压住食指/刀柄侧面；
9. 左手可以做防守或平衡，不要复制右手姿势；
10. 用 Knife Contact 再单独制作命中关键帧。

## 12. 朗基努斯之枪双手突刺

把它当作红缨枪/长矛，而不是超大单手手枪：

1. 载入“朗枪突刺命中 / Lance Contact”；
2. 先用 `lance` 把枪轴对准身体正前方；
3. 后手（通常右手）握在靠近身体的后段；
4. 前手（通常左手）握在更靠前的位置；
5. 两手沿枪杆分开，不能重叠，也不能隔空；
6. 后肘屈曲并靠近身体；
7. 前臂沿枪杆方向伸出，但不要把肘完全锁死；
8. 胸腔和骨盆有少量同向扭转；
9. 前腿承重、后腿蹬伸，枪尖—双手—胸腔形成清楚的力线；
10. 头仍然朝枪尖方向；
11. 从背面确认双臂没有反向交叉到背后；
12. 最后调手指包住枪杆。

准备帧和命中帧要分别导出。Codex 会在两帧之间制作蓄力、推进和回收。

## 13. 插入栓与背部入口

1. 载入站立姿势；
2. 打开插入栓与舱盖；
3. 选择“武器/插入栓骨骼”；
4. 单选 `entry_plug`；
5. 从侧面确定插入栓长轴与背部入口方向一致；
6. 从背面确定它位于身体中线；
7. 通过局部平移控制进入深度；
8. 不应在最终闭合状态露出大段白色/橙色栓体；
9. 分别调 `plug_hatch_l/r`，让舱盖在闭合时覆盖入口；
10. 需要动画时分别导出“完全拔出”“半插入”“完全插入”三个姿势。

目前插入栓来源模型未必完全匹配 Tiger 初号机背部，因此这里允许调整比例之外的
位置与角度；但先不要修改初号机主体网格。

## 14. 保存、导出与截图的区别

### 保存 Pose Lab

“保存 Pose Lab”或 `Ctrl+S` 保存可继续编辑的 `.blend`。它不会自动修改 Minecraft
资源包。

### 导出当前动作 JSON

1. 在“动作名”输入英文名，例如 `crouch_human_v1`；
2. 点击“导出当前动作 JSON”；
3. 输出目录：

```text
D:\eva\external-assets\work\pose-lab\exports\
```

每次会产生：

- `*.animation.json`：游戏骨骼旋转/位置；
- `*.animation.meta.json`：Blender 版本、源文件哈希、当前武器和坐标约定。

导出不会覆盖正式 `eva_unit01.animation.json`。告诉 Codex 动作名后，由 Codex 检查
差异、合并到正确动画和关键帧，再构建资源包。

### 输出五视图 PNG

会输出：

- `front.png`；
- `side.png`；
- `back.png`；
- `high_3q.png`；
- `pilot.png`。

目录：

```text
D:\eva\external-assets\work\pose-lab\captures\<姿势_时间>\
```

五张高分辨率渲染可能需要 1–3 分钟，期间 Blender 看起来像暂时无响应，不要连续
点击按钮。完成后底部状态栏/系统控制台会给出目录。

## 15. 撤销与故障恢复

- 撤销一步：`Ctrl+Z`；
- 重做：`Ctrl+Shift+Z`；
- 清除所选骨骼旋转：`Alt+R`；
- 清除所选骨骼位移：`Alt+G`；
- 重新载入正式动作：选择预设后点“载入姿势”；
- 模型只剩手脚：关闭“驾驶员视角遮挡规则”；
- 武器不见了：在“武器”中重新选择；
- 骨骼不见了：在 Outliner 选 `SEELE_Unit01_Rig`，再进入 Pose Mode；
- 误入 Edit Mode：立刻按 `Tab` 返回，不要保存骨架结构修改；
- 左右搞混：记住 `_l` 是 EVA 自己的左侧，正面看位于屏幕右侧；
- 模型彻底改乱：先确认重要动作已导出，再运行 `rebuild_pose_lab.bat`。

## 16. 人工与 Codex 的协作闭环

每个姿势按以下格式交接即可：

```text
动作：crouch_human_v1
人工结论：正面通过 / 侧面通过 / 驾驶员视角暂未通过
仍有问题：左脚脚尖略抬；右手腕需再向内 3–5 度
```

Codex 接手后负责：

1. 找到最新导出的 JSON；
2. 做骨骼名、轴向、数值范围与源哈希检查；
3. 合并到正式 GeckoLib 动画；
4. 生成离线五视图；
5. 构建资源包和 Mod；
6. 更新桌面测试启动文件；
7. 交回游戏内肉眼验收点。

人工只需要负责最有价值的部分：关节是否像人、轮廓是否合理、武器是否真的被握
住。代码接线、格式转换、构建和回归由 Codex 处理。
