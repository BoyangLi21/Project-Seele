# EVA 舰队机库、出击与回收测试

本文档定义 Unit-00、Unit-01、Unit-02 在连续 GeoFront 地图中的舰队后勤验收标准。这里的“完成”表示服务器状态、实体位置和方块路线满足合同；机库比例、灯光、按钮易读性与机械动作观感仍需玩家在游戏内逐项目视确认。

## 核心规则

- 一个世界中 Unit-00、Unit-01、Unit-02 **每种型号只能存在一台**。三台机体分别拥有自己的规范 UUID、湿式机库、运输轨道、发射井和地表站台。
- EVA 日常状态是停在对应湿式机库，不是长期停在发射井。湿式机库中的 LCL 液面应到肩部附近，肩部高度必须有可供玩家通行、观察和登机的实体栈桥。
- 正常出击路线必须是：湿式机库排放 LCL → 打开机库闸门 → 机械载台沿实体轨道移动到对应地下发射井 → 522 格实体竖井上升 → 到达东京-3 对应地表站台。
- 正常回收路线必须是：EVA 回到自己型号的地表站台并完全静止 → 地表回收指挥台操作员按下对应机号按钮授权 → 机械载台沿同一实体竖井下降 → 沿轨道返回对应湿式机库 → 关闭闸门并重新灌注 LCL。
- **站上地表站台不会自动回收。** 回收必须同时满足“正确机型、正确站台、机体静止、指挥台对应按钮获得授权”。错误站台、仍在移动、正在战斗或运输状态不允许开始回收。
- 出击和回收全过程不允许用维度切换、复制实体或瞬移伪造。正常流程中机体 UUID、驾驶员和所在维度必须保持不变。
- `/seele eva reset` 只用于损坏存档抢救，是管理员例外，不属于正常机械运输，也不得代替地表按钮回收。它可以从任意位置强制删除该型号的异常实例，并把唯一规范实例重建到对应湿式机库。

## 三座独立湿式机库

三台 EVA 必须分别停放，不能把不同型号叠在同一机库或共用同一个水槽：

| 机型 | 地下发射井 X | 日常停放 | 地表回收站台 |
|---|---:|---|---|
| Unit-00 | `2` | 00 号湿式机库 | 00 号站台 |
| Unit-01 | `30` | 01 号湿式机库 | 01 号站台 |
| Unit-02 | `58` | 02 号湿式机库 | 02 号站台 |

每座机库人工检查：

1. EVA 直立停在独立保持架内，型号与机库编号一致。
2. LCL 是 `projectseele:lcl` 流体，液面到机体肩部附近，不是橙色玻璃或普通水。
3. 肩部高度有连续、带护栏的观察/登机栈桥；玩家能从公共上层走廊安全走到机体插入栓入口附近，不能开门即坠入深井。
4. 控制按钮、标牌和观察席必须全部安装在实体平台上。按钮背后或下方必须有支撑方块，玩家脚下必须有完整地板；任何按钮浮在空中都视为失败。
5. 每台机体的准备按钮只控制本机。按下后应先看到 LCL 逐层排放，再看到闸门打开和载台移动，不能一按按钮就让 EVA 消失并在发射井出现。

## 最短出击验收

1. 首次或地图修复后执行：

   ```mcfunction
   /seele geofront setup
   /seele geofront audit
   /seele eva reset all
   /seele geofront hangar
   /seele eva status all
   ```

2. 确认三台机体都报告为机库停放状态，并各自处于正确湿式机库。
3. 玩家从肩部栈桥进入待测 EVA。由另一位玩家在机库上层控制平台按该机型的准备按钮；单人确定性测试可用 `/seele eva prepare unit01`，再分别测试 `unit00` 与 `unit02`。
4. 观察 LCL 排放、闸门打开、载台沿实体轨道进入对应发射井。运输过程中机体应被冻结，不能走下载台或自行驾驶穿墙。
5. 发射联锁完成后，载台与原 EVA、原驾驶员沿 522 格竖井上升。中途用 `F3` 确认维度始终为 `projectseele:geofront`；用 `/seele eva status <机型>` 确认阶段变化。
6. 到达地表后检查机体位于自己的站台，而不是相邻机型站台；机体恢复驾驶控制，UUID 与出发前一致。

## 地表指挥授权回收

地表三座站台旁必须有一个可步行抵达的回收指挥平台。平台上至少有 00、01、02 三个明确标号的实体按钮，并满足：

- 每个按钮都固定在指挥台或墙面上，操作员脚下有实体地板；不得悬空。
- 按钮和站台之间有安全通路，不要求操作员跳入竖井，也不能让操作员站在移动载台上操作。
- 每个按钮只授权对应机型；按错机号必须拒绝并给出可读提示。
- EVA 尚未回到对应站台、未完全静止或状态不是已部署时，按钮必须拒绝回收。

人工流程：

1. 驾驶 EVA 回到对应地表站台中央，松开移动、跳跃和攻击输入，让机体完全静止。
2. 保持 EVA 在站台上；仅仅等待不能触发回收。
3. 由地表指挥中心的玩家按下该机号实体回收按钮。正常回收没有远程命令旁路；单人测试也必须实际走到指挥台并按按钮。
4. 授权通过后，机体和载台沿原 522 格竖井物理下降。下降过程中检查 UUID、驾驶员和维度不变，输入处于机械联锁状态。
5. 到达地下井后，载台沿实体轨道移回该机型湿式机库。应按“载台到位 → 闸门关闭 → LCL 逐层灌注”的顺序结束。
6. `/seele eva status <机型>` 最终应再次报告为机库停放状态。前往肩部栈桥，确认 LCL 恢复到肩部附近且 EVA 没有残留在发射井或地表。

负面测试至少各做一次：

- EVA 停到错误机号站台后按回收按钮：拒绝。
- EVA 正在移动时按对应回收按钮：拒绝。
- EVA 不在站台时按对应回收按钮：拒绝。
- 只把 EVA 停在站台并等待、不按按钮：不得自动下降。
- 连续快速点击按钮：只能启动一次流程，不能复制 EVA 或载台。

## 强制重置与世界唯一性

分机型抢救命令：

```mcfunction
/seele eva reset unit00
/seele eva reset unit01
/seele eva reset unit02
```

三机一起抢救：

```mcfunction
/seele eva reset all
```

重置验收：

1. 先记录 `/seele eva status all`，确认待测机型当前阶段和规范 UUID。
2. 在地表、战区或运输异常位置执行对应分机型重置命令。
3. 若有人驾驶，必须先安全卸载驾驶员；不能把玩家埋入墙体、LCL 池底或虚空。
4. 旧的已加载同型号实例必须移除，只在对应湿式机库生成一个新的规范实例，并恢复机库停放状态。
5. 再次执行 `/seele eva status all`，确认 00、01、02 各一台；其他两个未重置机型不应被移动或更换 UUID。
6. 尝试用召唤命令制造同型号第二台。服务器应拒绝非规范重复体进入世界，重新登录或重载区块后也不能出现重复体。

这里的“一世界一型号一台”是全世界范围的规则，不是只检查玩家附近。未加载区块中的旧重复体在再次加载时也必须被唯一性门禁拒绝；管理员重置不能留下第二台隐形或远距离实例。

## 验收记录

建议每次人工测试记录：

- 存档名与模组提交号；
- 测试机型；
- 出发前、地表到达后、回库后的 UUID；
- 出击和回收各阶段的 `/seele eva status` 输出；
- 三座湿式机库、肩部栈桥、地表回收指挥台的截图；
- 所有按钮是否真正落在平台/控制台上；
- 是否发生瞬移、复制、错井、错站台、驾驶员脱离、卡墙或悬空按钮。

自动编译和结构审计只能证明状态机与坐标合同存在，不能证明机库比例、LCL 肩线、按钮高度、通路安全和机械动作观感已经合格；这些项目必须由玩家肉眼验收。
