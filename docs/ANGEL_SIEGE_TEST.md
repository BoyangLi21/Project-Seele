# NERV 信标使徒围城测试

该事件是 Phase 3 的多人防卫玩法切片：玩家部署一座 NERV 信标，Sachiel、Shamshel、Zeruel 分三波从 92 格外进攻。当前几何模型仍是待美术验收的原型；本页只验收服务端状态、A.T. Field、重启恢复、区块加载、信标攻防和奖励闭环。

## 当前数值（新机制，待玩家拍板）

| 项目 | 当前值 |
|---|---:|
| 信标结构耐久 | 1200 |
| Sachiel 对信标伤害 | 45/秒 |
| Shamshel 对信标伤害 | 30/秒 |
| Zeruel 对信标伤害 | 75/秒 |
| 波次时间 | 5 秒 / 30 秒 / 55 秒 |
| 生成距离 | 92 格 |
| 胜利奖励 | S² 机关碎片 ×3 |

这次没有改动三种使徒原有 HP、护甲、A.T. Field、对玩家/EVA 伤害或攻击冷却；上表只是此前不存在的信标结构耐久机制。

## 单人功能测试

1. 在创造模式执行 `/give @s projectseele:nerv_beacon`，选择离建筑至少 120 格的普通地形。
2. 右键地面放置信标。应生成 3×3 铁基座、磁石和避雷针，并显示“波长蓝色”。
3. 5 秒后 Sachiel、30 秒后 Shamshel、55 秒后 Zeruel 分别从不同方向出现；三者没有玩家/EVA 目标时仍必须朝信标推进，不能停在未加载区块。
4. 普通枪械和阳电子炮不得穿透未中和的 A.T. Field；EVA 近战可消耗力场。该规则与 Ramiel 一致。
5. 击杀全部三波。在线部署者获得 S² 机关碎片 ×3；部署者离线时奖励实体落在磁石上方，不得静默丢失。
6. 故意不防守另一座新信标。使徒靠近后每秒破坏结构耐久；归零时磁石和避雷针消失，事件失败。

## 重启恢复测试

1. 放置信标并等第一波生成。
2. 正常保存退出；专用服务器执行 `stop`，不要强制杀进程。
3. 重新进入同一世界。事件必须从世界 `SavedData` 恢复，而不是重新从第 0 tick 开始：
   - 已生成使徒保留 UUID 和信标锚点；
   - 下一波仍按最初开始时间到达；
   - 信标剩余耐久保持；
   - 至多约 22 个沿三条进攻路线的区块被有限强加载，事件结束后全部释放。
4. 删除磁石或把耐久打空后重启，事件不得复活。

## 指挥室联动与管理员命令

1. 围城开始后，在作战指挥室查看战略状态屏；屏幕必须显示 `NERV BEACON`、当前波次、结构耐久和存活使徒数。
2. 执行 `/seele siege status`；聊天栏摘要必须与战略屏一致，附近没有事件时明确报告 `INACTIVE`。
3. 仅在测试清场或事件卡死时执行 `/seele siege abort`。该命令应移除本事件追踪的使徒、释放区块票、删除持久化事件，但不能影响地图上的其他普通实体。
4. 重新部署信标后，战略屏必须从新事件的 `WAVE 0/3` 与 `1200/1200` 开始，不能残留上一次状态。

## 专用服务器前置

`gradlew.bat runServer` 已能在服务端环境完成模组发现并到达 EULA 边界。第一次真正启动前，服务器所有者必须亲自阅读 `eula.txt` 链接并把 `eula=false` 改为 `eula=true`；自动化不会代替用户接受 EULA。

## 离线契约

```bat
python tools\validate_angel_siege_contract.py
```

该脚本验证 SavedData 版本、波次目录、锚点 NBT、有限区块票、结构耐久、离线奖励和清理路径。它不能代替玩家对导航、攻击节奏和三种原型模型的肉眼验收。
