# TV 版 EVA 驾驶视角与插入栓设计基线

> 状态：研究与实现约束，不代表游戏内视觉已经验收。
> 本文只记录文字结论、公开网页链接和原创实现建议；仓库不保存官方截图、动画画面、台词录音或 UI 临摹素材。

## 1. 先给结论

Project SEELE 的第一人称不应该被理解成普通 FPS 的“两只手固定浮在屏幕下方”，也不应该把驾驶员真的放到 EVA 头部、从模型内部向外看。

TV 版更接近下面这套结构：

1. 驾驶员坐在插入栓内，插入栓位于 EVA 颈后至躯干的脊柱区域；驾驶员的物理位置不等于 EVA 的眼睛位置。
2. 插入栓注满 LCL，完成连接和视觉回路后，驾驶员通过环绕显示获得接近无遮挡的 EVA 周围视野。
3. 神经同步让驾驶员把 EVA 当成自己的身体，传递动作意图、姿态感觉、冲击和疼痛；双手控制杆仍承担武器和精细手部操作。
4. 因此游戏画面应由“EVA 眼位的外界画面 + 插入栓内的固定边缘/控制器 + 状态式 HUD + 同步反馈”组成。
5. 画面中的 EVA 手臂、手和武器必须来自**同一台第三人称实体、同一副骨骼、同一动画时刻**。只有当真实肢体进入 EVA 眼睛视野时才应出现，不能另造一对独立 FPS 手臂。

这也解释了用户要求的“第一、第三人称统一”：第一人称不是复制第三人称全身模型摆到屏幕前，而是从同一世界姿态的 EVA 眼位观看。姿态同步是硬约束，画面构图则由眼位和视锥自然决定。

## 2. 资料结论与可信度

### 2.1 插入栓、LCL 与全景显示

- [EvaGeeks：Entry Plug](https://wiki.evageeks.org/Entry_Plug) 将插入栓描述为插入 EVA 脊柱区域的胶囊式驾驶舱；LCL 透明化后，内壁显示器为驾驶员提供几乎无遮挡的三维外界视野。该页面还指出，主显示中的“盲区”实际是 EVA 自身身体轮廓。这是当前“同一实体进入自身视野”方案最重要的依据。
- [EvaGeeks：LCL](https://wiki.evageeks.org/LCL) 记录 TV 连续性中的 LCL 为琥珀/橙色透明液体，启动时经处理后视觉上接近空气，但仍是液体；偶发气泡可以提示这一点。
- [EvaGeeks 第 14 集场景索引](https://wiki.evageeks.org/FGC:Episode_14_Scene_09) 逐镜记录了插入完成、第一次连接、LCL 处理、数据确认与第二阶段接触等启动步骤，可作为启动演出的顺序参考，但不应复制剧中文字版式或台词。

可信度说明：EvaGeeks 是整理 TV 正片的二手资料，并非 khara 官方设定站；上述结论可作为结构基线，细节仍需以合法观看的正片和正式设定资料人工复核。

### 2.2 驾驶席、双控制杆与 HUD

- [EvaGeeks：Entry Plug 的驾驶席与显示章节](https://wiki.evageeks.org/Entry_Plug#Pilot's_seat_and_display) 记录驾驶员坐在大型座椅组件中，主要手动控制来自 induction levers 及其按钮，其他信息通过按需出现的全息 HUD 提供。
- [Evangelion Wiki：Entry Plug](https://evangelion.fandom.com/wiki/Entry_Plug) 的跨版本整理称标准驾驶席有左右两只带扳机的控制装置；其资料还区分初号机较连续的显示面和二号机更明显的正面/侧面分区。该站为爱好者维基，只适合辅助确认，不作为唯一依据。
- [EVA GLOBAL 官方商品页：Entry Plug Interior](https://evaglobal.jp/en/figure/asuka002/) 明确提到可动控制杆和可替换 HUD。这是官方页面，但商品表现的是新剧场版，而不是 TV 版；只用来确认“控制杆 + 抬头显示”这类共通硬件，不据此把新剧场版造型混入 TV 版目标。

### 2.3 同步、身体感与损伤反馈

- [Wikipedia：Evangelion (mecha)](https://en.wikipedia.org/wiki/Evangelion_%28mecha%29) 汇总了 LCL、A10 神经与同步接口，并引用多种出版资料。它支持“驾驶员与 EVA 是神经/情绪层面的身体连接”这一大方向，但具体界面造型不能从百科推导。
- [EvaGeeks 第 3 集场景索引](https://wiki.evageeks.org/FGC:Episode_03_Scene_07a) 记录了断电倒计时、警报、碰撞后驾驶员状态和指挥通信。可据此把电源、损伤和外部通信做成事件驱动反馈，而不是常驻一整屏仪表。

### 2.4 瞄准与枪械操作

- induction lever 的资料倾向于说明：EVA 的全身动作依赖神经同步，枪械、扳机和精细手部动作仍需要手动控制器参与。
- [EvaGeeks：Entry Plug](https://wiki.evageeks.org/Entry_Plug) 记录了信息 HUD 按需出现；[TV 第 3 集场景索引](https://wiki.evageeks.org/FGC:Episode_03_Scene_07a) 记录剩余电量显示会在断线时进入倒计时。这种“状态触发，而非常驻驾驶舱仪表墙”的节奏更接近 TV 版观感。
- 长距离阳电子炮可以使用专用光学/射击界面，但 Project SEELE 应原创网格、字形和提示逻辑，不逐像素临摹第 6 集画面。

## 3. 两种“第一人称”必须分清

### 3.1 驾驶员在插入栓里看到的界面

这是最终交给玩家的画面：

- 外界影像来自 EVA 头部/视觉系统的视点；
- 视野外围保留少量驾驶席、插入栓内壁和双控制杆轮廓；
- LCL 不是永远盖住屏幕的橙色滤镜，启动后应主要表现为轻微折射、漂移微粒、气泡和边缘光；
- HUD 是覆盖于全景显示上的信息层，在武器、损伤、断电或同步异常时增强；
- 通信、目标锁定和警报按事件出现，平时尽量让玩家看到战场。

### 3.2 EVA 眼睛的第一人称感知

这是世界画面的来源：

- 摄像机锚点属于 EVA 的头/眼位，而不是驾驶员实体的头；
- 身体轮廓是同一台 EVA 的世界模型。低头、侧看或摆臂时，真实肩、臂、手和武器可能进入画面；
- 站立、蹲伏、趴行、持刀、持枪和受击都只使用第三人称同一动画状态；
- 摄像机旋转不能凭空带动一对独立手臂。视线瞄准若影响上身，必须通过同一套头、脊柱、肩、臂瞄准层反映到第三人称；
- 近裁剪面应避免切出骨盆、无头躯干或漂浮断臂，但不能靠另画假手掩盖问题。

### 3.3 为什么普通 FPS 手臂方案不适合

普通 FPS 会为画面专门制作夸张、缩短和偏移的 viewmodel；它允许第一人称和世界模型动作不同。用户现在要求的恰好相反：

- 第一人称刀、枪、空手动作要和别人看到的动作一致；
- 两只手不能独立漂浮或一起镜像摆动；
- 手臂位置由真实肩关节、肘关节和武器握点决定；
- 视线中看不到手并不一定是错误——若站立自然下垂，手本来可能在视锥下方；真正错误是为了“必须看到手”而把肩臂折到身体后面或悬在画面中央。

所以设计目标不是“任何时候都看见两只手”，而是“任何看见的身体部分都与第三人称完全相同，并且插入栓反馈始终让玩家感到自己在操纵这具身体”。

## 4. 对现有 `EvaHud` 的审视

现有实现已经有可保留的状态基础：

- 六秒插入演出：机械锁定、LCL 上升、A10 连接、同步启动；
- 实时同步率；
- 机体血量、AT Field、当前武器与姿态；
- 发射联锁/上升状态；
- 阳电子炮充能和专用瞄准层；
- 受伤红闪。

但当前视觉逻辑更像“Minecraft 机甲模组 HUD”，离 TV 版插入栓感还有这些差距：

1. 常驻矩形状态面板过重，驾驶舱空间感太弱；应把重要数据拆成外围、半透明、按需出现的显示。
2. 橙色四边框只能表达 UI，不能表达驾驶员被包在圆筒形插入栓中；需要下方座椅/控制器和侧面内壁的稳定轮廓。
3. `HDG/PITCH` 和人工地平线属于 Project SEELE 的原创可用功能，不应写成“TV 原作还原”。它们可以保留，但要降低常驻权重。
4. LCL 当前主要是填充色和扫描线，缺少透明液体中的折射、漂浮、压力和声音反馈。
5. 受击只有红闪，缺少“同步身体受到冲击”的方向、重量、短暂失真和音频反馈。
6. HUD 与真实武器握持/枪口/瞄准轴尚未形成同一数据链。准星不能只等于屏幕中心，应该表示武器当前实际可达的命中方向。

## 5. 原创驾驶舱视觉方案

### 5.1 画面分层

从后到前固定为：

1. **世界层**：EVA 眼位摄像机渲染战场和同一 EVA 可见肢体。
2. **全景显示层**：极轻的显示面拼缝、暗角、受损噪声和传感器失真；不遮挡战斗主体。
3. **物理驾驶席层**：屏幕下侧左右各一只原创控制杆根部，侧边少量舱壁轮廓。它们是驾驶员手边硬件，不是 EVA 的手。
4. **战术 HUD 层**：同步、能量、AT Field、目标和武器信息，按状态显隐。
5. **生理/LCL 层**：微弱橙色边缘折射、少量气泡、冲击波纹和同步异常脉冲。

这五层不能混为一对漂浮 EVA 手臂。

### 5.2 平时画面

- 中央不常驻大型十字准星；空手/粒子刀只保留很淡的注视点。
- 同步率放在上方窄弧或侧上方小读数，稳定时降低亮度。
- 血量不用写成“HP”；使用原创的机体完整性分段图或 `BODY STATUS`。
- AT Field 用环形/六边形脉冲和颜色表达“展开、受压、恢复”，数字为辅助。
- 下方控制杆只露出根部、握柄内侧和指示灯，不占用中央视野。
- 初号机显示面可以较连续；二号机可以用更明确但原创的侧屏分区；零号机偏原型/测试读数。差异应来自布局参数，不复制官方画面。

### 5.3 枪械/阳电子炮模式

- 切换到枪械时，左右 induction lever 进入武器操作状态：握柄指示灯亮、扳机保险解除、两侧出现握持确认。
- 准星分两层：视线意图点与真实枪口投射点。未对齐时两点分离，对齐后合并，直接暴露第三人称持枪姿势是否正确。
- 显示 `MUZZLE ALIGN`、距离、蓄能、散热和目标遮挡；文案与字体版式全部原创。
- 命中规则直接提示：AT Field 完整时显示 `FIELD BLOCK`；核心暴露才显示核心有效锁定。不要用准星颜色掩盖实际射线/枪口不一致。
- 阳电子炮蓄能时逐步收窄视场，但必须保留少量插入栓外围，避免突然变成完全独立的狙击游戏界面。

### 5.4 刀、长枪与近战

- 粒子刀不需要枪械准星；显示左右手、武器握点和目标距离是否进入突刺/挥砍范围。
- 朗基努斯枪双手持时，HUD 的中轴提示应来自枪身与枪尖方向；第一人称能看到的枪必须是第三人称同一武器骨骼。
- 近战蓄力用控制杆压力条、同步脉冲和肩/上身准备动作表达，不添加一套第一人称专属挥刀动画。

## 6. 原创声音与运动反馈清单

所有声音必须原创合成或来自许可明确的 CC0 素材，并登记在 `docs/ASSETS.md`。不得截取动画音轨、原版警报、角色台词或 BGM。

| 状态 | 原创声音建议 | 运动/视觉建议 |
|---|---|---|
| 舱门锁定 | 多层金属闩、低频壳体共振 | 短暂黑场，舱壁闭合 |
| 插入栓下降/旋入 | 液压、导轨、锁销逐级到位 | 画面轻微纵向惯性，不旋转玩家视角 |
| LCL 注入 | 过滤泵、液体涌入、近距离气泡 | 液面自下而上，短暂折射增强 |
| LCL 处理 | 柔和电流扫频、气泡骤减 | 橙色浑浊逐步透明，显示面启动 |
| A10/神经连接 | 分段脉冲和稳定低鸣 | 周边神经图形逐段接通，避免复制官方字样 |
| 同步稳定 | 与同步率相关的低频呼吸感 | 摄像机与身体动作的迟滞逐步减小 |
| 行走 | 远低于普通脚步的结构低频、装甲细响 | 每步只给小幅垂直冲量和质量延迟 |
| 冲刺 | 加快的伺服和风压 | 轻微前倾、视野边缘速度感 |
| 起跳/落地 | 肌体发力、地面冲击和舱体回弹 | 起跳压缩、空中惯性、落地一次性冲量 |
| 蹲伏/趴行 | 关节负载、地面摩擦 | 摄像机高度连续过渡，不瞬移，不丢失身体 |
| 武器切换 | 控制杆伺服、保险拨片 | HUD 只在切换时展开，稳定后收拢 |
| 受击 | 方向性重击 + 舱内低频闷响 | 短方向冲量、LCL 波纹、信号噪声；禁止持续乱晃 |
| 同步异常 | 不稳定差频、心跳式脉冲 | 肢体反馈延迟、双影、色差，可由无障碍选项关闭 |
| 内部电源倒计时 | 原创节拍报警、频率随余量提高 | 只在断线后出现，剩余时间获得最高视觉优先级 |

### 无障碍限制

- 镜头震动、色差、闪光、LCL 折射和警报音量必须分别可调。
- 不用高频全屏闪烁表达同步或损伤。
- 摄像机运动必须有上限，受击后快速回中，不能夺走瞄准控制。

## 7. 第一、第三人称统一的实现合同

本节是后续 Java/动画工作的验收合同，不是当前完成声明。

1. **唯一权威姿态**：实体动画控制器产出一副世界骨架姿态；站立、走、蹲、趴、刀、枪、长枪都从这里取值。
2. **唯一武器挂点**：武器只挂到正式 socket；第一人称不可另算一个位置。
3. **唯一时间源**：同一 tick、partial tick、动画状态和瞄准参数用于两种视角。
4. **眼位相机**：摄像机取头/眼 socket 的世界变换，但不把相机作为骨骼父节点反过来拖动手臂。
5. **真实自体可见性**：由摄像机渲染同一实体的可见肢体子树，或使用完全等价的同骨架第二遍渲染；不得对左右臂分别施加画面偏移。
6. **遮挡规则**：隐藏会包住摄像机的头部内壳和近处背面，不隐藏为了修错位而应该出现的肩、臂和武器。
7. **瞄准链**：输入先得到目标方向，再由上身/双臂 IK 或受限瞄准层去对齐武器；最终命中射线从真实枪口 socket 发出。
8. **控制杆独立**：驾驶舱下方两只控制杆属于 cockpit overlay/model，不属于 EVA 骨骼，也不冒充 EVA 的手。

最关键的一条是：**相机跟随身体，身体不跟随相机专用假手。**

## 8. 人工视觉验收表

每次只验证一个动作，并同时提供同一 tick 的第三人称正/侧视与第一人称：

### 空手站立

- 侧视图中上臂没有整体翻到躯干后方；肩、肘、腕形成自然下垂线。
- 第一人称没有漂浮断手、骨盆、无头躯干或左右手粘连。
- 若手臂未进入眼位视锥，允许看不到手；低头后应能看到与第三人称一致的胸肩/臂轮廓。

### 走、蹲与趴

- 第一人称摄像机高度连续变化；姿态切换过程中不能整套身体消失。
- 两臂的前后支撑关系与第三人称相同，趴行不得变成游泳或手背到身后。
- 相机左右观察不会让两条胳膊同步绕屏幕中心旋转。

### 粒子刀

- 刀只出现在持刀手的真实 socket。
- 第一、第三人称在准备、接触和回收关键帧的持刀手、空手和刀尖方向一致。
- 不使用第一人称专属刀模型或专属挥砍时序。

### 枪械/阳电子炮

- 第三人称双手都确实接触武器，左手不向外上方折断。
- 视线意图点、枪口投射点与实际命中点能被调试层同时显示。
- 第一人称看到的是同一把枪和同一双手，不因切换武器恢复旧模型或旧动画。
- 趴射必须单独验收肩、肘、枪托、枪身与地面的关系。

### 朗基努斯枪

- 双手持握点固定；突刺过程中前手导向、后手推进。
- 第一人称枪尖方向与第三人称完全一致，命中检测从同一枪尖/武器轴计算。

## 9. 推荐实施顺序

1. 先修三台 EVA 空手 idle 的肩臂方向，确认模型坐标系和正前方。
2. 建立同骨架、同 tick、同武器 socket 的第一人称世界观察，暂时关闭所有独立 FPS 手臂偏移。
3. 依次完成粒子刀、双手枪、双手长枪；每个动作单独人工通过后再做下一个。
4. 再加入原创插入栓物理边缘和双控制杆，明确它们不是 EVA 手臂。
5. 最后重排 `EvaHud`，让同步、AT Field、电源、武器和目标信息按状态出现。
6. 加入原创 LCL、音效、惯性和同步异常反馈，并提供无障碍开关。

在第 1—3 项人工通过以前，不应靠增加 HUD、滤镜或更多内容掩盖骨骼与武器姿态问题。
