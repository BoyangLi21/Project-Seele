# EVA 武器柜与实体装载测试

Phase 2.4 不再把 EVA 武器只当成 R 键枚举。地图 schema v17 会在三座地下发射笼左侧各放置一台 `NERV EVA 武器柜`；新柜首次生成时预装粒子刀、帕雷特步枪、阳电子炮、朗基努斯之枪和 N² 自毁装置各一件。柜内五格库存、轮换位置以及 EVA 当前武器/装载掩码都会跨重启保存。

## 旧存档升级

1. 关闭游戏并保留存档备份。
2. 用最新构建进入 `SEELE_TOKYO3_REBUILT`。
3. 执行 `/seele geofront audit`。输出必须包含 `valid=true mapVersion=17`。
4. 如果地图从未完成初始化，才执行 `/seele geofront setup`；不要对已经完整的地图反复 setup。
5. 三座地下发射床左侧 10 格处应各出现一台深色、橙色识别条的武器柜；右侧仍是脐带供电柱。

旧地图只补缺失的柜。已经存在但被玩家取空的柜不会因登录或 audit 自动补货，避免无限复制武器。

## 操作语义

武器柜只搜索 48 格内最近的存活 EVA。机体必须供电、静止，并且不处于发射、启动锁、暴走、强制停机或十字束缚状态。

- 手持一种 EVA 装备右键：把 1 件放入柜中；创造模式不消耗手持物。
- 空手右键：从柜中按持久化顺序取下一件装备，自动把机体原武器归还柜中，再为最近 EVA 装上新武器。
- 空手 Shift+右键：如果最近 EVA 正在持武器，先卸回柜中并恢复空手；否则从柜中取一件到玩家物品栏，物品栏满时掉在脚边。
- 破坏武器柜：柜内物品全部正常掉落。

N² 是不可逆战略武器。仅仅装上不会爆炸；仍需驾驶时切到 N² 并连续按住使用键完成原有解锁流程。

## 兼容模式与严格模式

默认配置：

```toml
[eva_armament]
requireRackLoadout = false
```

默认关闭是为了让现有动作/武器视觉测试仍能用 R 键访问全部六种状态。将它改为 `true` 并重启服务器后：

- R 键只会遍历空手和武器柜实际装入该机体的那一件装备；
- 柜中换装会同时更新机体的装载掩码；
- 卸载后 R 键只能保持空手；
- 配置切回 `false` 后完整测试轮盘立即恢复，但物理装载记录不会丢失。

## 人工验收清单

1. 对 01 柜空手右键五次，确认五类装备依次出现且上一件回柜，库存总量不凭空增加。
2. 装粒子刀后退出世界再进入，确认机体仍持粒子刀；柜内其余四件仍在。
3. Shift+右键卸刀，重登后确认机体仍为空手、刀已回柜。
4. 开启严格模式，装帕雷特步枪；驾驶后连续按 R，只应在空手/步枪之间切换。
5. 让机体移动或进入发射流程时操作柜，必须收到拒绝提示且柜内库存不变化。
6. 破坏一台有五件库存的柜，地面应出现五件装备和柜本体；重新执行 audit 不应给已被玩家主动拆除后的现场凭空刷武器，只有 setup/地图修复流程才负责基础设施重建。
7. 专用服务器停服再开，重复 2–4，确认服务端权威结果与两客户端一致。

自动门禁：

```bat
python tools\validate_eva_armament_contract.py
```

自动检查只证明注册、存档、交换、地图升级和资源契约存在；握持姿势与柜体观感仍由人眼验收。
